1 OPTIONS

1.1 -r

–recursive
对于HTTP主机，wget首先下载URL指定的文件，然后（如果该文件是一个HTML文档的话）递归下载该文件所引用（超级连接）的所有文件（递归深度由参数-l指定）
对FTP主机，该参数意味着要下载URL指定的目录中的所有文件，递归方法与HTTP主机类似。

1.2 -l depth

–level=depth
指定递归的最大深度

1.3 -k

–covert-links
转化文档中的链接为本地链接

1.4 -O file

–output-document=file
保存文件名

1.5 -o logfile

–output-file=logfile
所有输出信息记录到logfile中

1.6 –limit-rate=amount

后跟限制下载速度

1.7 -c

–continue
续传（下载一个大文件时突然发生中断，可以继续下载）

1.8 -b

–background
后台下载，产生wget-log文件显示日志

1.9 –spider

后跟下载链接用来测试链接是否可用

1.10 -t number

–tries=number
指定重试次数

1.11 -T seconds

–timeout=seconds
超过seconds则停止

1.12 -q

–quiet
关闭输出

1.13 -v

–verbose
详细输出，默认

1.14 -i file

–input-file=file
下载链接在file文件中

1.15 -B URL

-base=URL
基址

1.16 -A acclist

–accept acclist

1.17 -R rejlist

–reject reglist
使用逗号分隔的文件后缀或者通配符列表

1.18 –user=user

1.19 –password=password

用户名和密码，可以用于ftp和http，–ftp-user=user –ftp-password=password –http-user=user –http-passwor

1.20 -p prefix

–directory-prefix=prefix
将下载文件保存在指定prefix中，默认是.即当前目录

1.21 -m

–mirror
下载事件网站 wget -m fanhan.tk

2 例子

2.1 下载网页

wget fanhan.tk
原封不动地下载该网站的index.html，但打开后发现由于路径问题链接的css路径 css/vf.css 不存在所以页面根本没法看，于是我们可以用-k选项将链接转换成适合在本地查看的形式，
wget -k fanhan.tk
此时可以看到 css/vf.css转变成了http://fanhan.tk/css/vf.css 于是打开index.html时浏览器会下载该文件以显示页面

2.2 下载网站

wget -m fanhan.tk
把整个网站都下载下来

2.3 下载文件并重命名

wget -O emacs.tar.gz http://ftp.gnu.org/gnu/emacs/emacs-24.1.tar.gz

2.4 下载所有匹配文件

wget -r -A “*.tar.gz" http://ftp.gnu.org/gnu/emacs
下载http://ftp.gnu.org/gnu/emacs 目录下的所有以tar.gz结尾的文件
